# AGENTS.md

## 本目录职责
- 维护 Face ID 相关识别能力与接入封装。
- 重点是识别流程、结果映射和与账户体系的衔接。

## 允许修改范围
- 允许修改本目录 provider 适配、请求封装、响应转换和必要的局部辅助函数。
- 新增供应商时优先沿用已有 provider 注册与选择模式，不要破坏统一接口。

## 必须复用的现有实现或组件入口
- 优先复用本目录已有 `provider`、默认实现、共享 HTTP 工具和错误包装方式。
- 跨层需要的数据结构优先复用 `object/`、`controllers/` 已有模型，不要另起一套字段命名。

## 禁止事项
- 不要把页面逻辑、控制器参数解析或无关业务决策塞到集成目录。
- 不要为了单个供应商复制一整套通用流程。

## 本目录最小验证命令
- 优先执行 `go test ./...`，至少确认本包和直接调用方能通过。
- 涉及外部接口常量或签名时，再补查受影响集成测试或调用点。

## 与上级规则的继承或覆盖说明
- 继承仓库根目录 `AGENTS.md`；本目录额外强调“优先守住统一 provider 接口”。
- 如果上级规则与供应商协议细节冲突，以不破坏统一抽象为先。
