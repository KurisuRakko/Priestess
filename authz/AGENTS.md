# AGENTS.md

## 本目录职责
- 维护鉴权模型与访问控制判断逻辑，负责把权限语义落到可执行判断上。
- 这里的改动通常会影响全局访问行为，优先保证规则可读性与兼容性。

## 允许修改范围
- 允许调整鉴权判断、权限辅助函数和与授权模型直接相关的小型封装。
- 如果变更会影响 API 权限语义，需要同步检查 `controllers/` 与 `object/` 的调用点。

## 必须复用的现有实现或组件入口
- 优先复用现有鉴权入口和已有权限判断模式，不要重新发明第二套授权判断。
- 跨层规则尽量复用 `object/` 中的对象能力，而不是在本目录硬编码资源细节。

## 禁止事项
- 不要把具体 HTTP 请求解析、页面逻辑或数据库细节塞进授权层。
- 不要通过复制 if/else 分支临时修补权限边界。

## 本目录最小验证命令
- `go test ./...`
- 如改动权限边界，至少补跑受影响资源的后端测试。

## 与上级规则的继承或覆盖说明
- 继承仓库根目录 `AGENTS.md`；本目录额外强调“全局权限影响优先评估”。
- 与上级冲突时，仅在授权语义层面以本目录规则为准。
