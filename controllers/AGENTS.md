# AGENTS.md

## 本目录职责
- 维护 HTTP 控制器、请求参数解析、权限入口和响应封装。
- 控制器层应保持瘦身，只负责把请求转交到领域层。

## 允许修改范围
- 允许修改控制器、局部辅助解析函数以及与接口直接相关的注释。
- 涉及复杂业务规则时优先下沉到 `object/` 或 `service/`，不要在这里堆逻辑。

## 必须复用的现有实现或组件入口
- 优先复用 `base.go`、`util.go`、同资源已有 CRUD/上传/校验模式。
- 新增接口前先搜索相邻资源是否已有近似实现和统一返回方式。

## 禁止事项
- 不要在控制器里写数据库细节、长流程编排或重复权限判断。
- 不要为了单个接口复制整套 controller 模板。

## 本目录最小验证命令
- `go test ./controllers/...`
- 如改动影响资源权限或业务流程，再补跑相关 `object` 包测试。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“控制器薄层化”。
- 若与上级冲突，优先把复杂逻辑迁回领域层。
