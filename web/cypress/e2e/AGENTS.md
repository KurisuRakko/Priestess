# AGENTS.md

## 本目录职责
- 维护端到端测试用例，覆盖登录、资源管理、权限、支付等真实用户流程。
- 这里的重点是业务路径覆盖与断言稳定，不是组件实现细节。

## 允许修改范围
- 允许新增或修改 Cypress 用例、流程断言和与页面行为直接相关的测试组织方式。
- 若测试暴露页面结构问题，修复页面逻辑应回到 `web/src`。

## 必须复用的现有实现或组件入口
- 优先复用已有登录流程、资源 CRUD 测试节奏和同资源测试文件命名方式。
- 新增流程优先在相近资源用例上扩展，而不是复制整份脚本。

## 禁止事项
- 不要把大量公共命令或选择器常量直接散落在每个测试文件。
- 不要把测试产物目录纳入源码逻辑。

## 本目录最小验证命令
- `cd web && npx cypress run --spec cypress/e2e/<file>.cy.js`
- 批量改动时执行 `cd web && npx cypress run`。

## 与上级规则的继承或覆盖说明
- 继承 `web/cypress/AGENTS.md` 与上级规则；本目录额外强调“用户流程断言优先”。
- 当前目录只覆盖用例层细则。
