# AGENTS.md

## 本目录职责
- 维护前端 MFA 相关子流程、组件与多因子交互细节。
- 重点是验证步骤清晰、错误反馈准确和流程可恢复。

## 允许修改范围
- 允许修改 MFA 组件、交互状态、局部样式和与多因子流程直接相关的前端逻辑。
- 需要请求支持时优先复用 `MfaBackend` 或认证主流程已有入口。

## 必须复用的现有实现或组件入口
- 优先复用 `web/src/auth` 的页面节奏和 `web/src/common` 的反馈组件。
- 新增因子交互前先参考现有启用、校验、扫码等模式。

## 禁止事项
- 不要在 MFA 子流程中复制完整登录页逻辑。
- 不要让局部状态机与主认证流程脱节。

## 本目录最小验证命令
- `cd web && yarn fix`
- 如改动认证关键流程，可补跑 `cd web && npx cypress run --spec cypress/e2e/login.cy.js`。

## 与上级规则的继承或覆盖说明
- 继承 `web/src/auth/AGENTS.md` 和上级规则；本目录额外强调“MFA 步骤清晰与恢复能力”。
- 当前目录只覆盖多因子子域。
