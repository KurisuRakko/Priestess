# AGENTS.md

## 本目录职责
- 维护前端 API 请求封装、请求过滤和后端资源访问入口。
- 这里负责让页面层不直接处理低层请求细节。

## 允许修改范围
- 允许新增或修改资源后端模块、请求参数封装和通用 fetch 过滤逻辑。
- 页面组件需要新接口时，默认先扩展这里，再由页面调用。

## 必须复用的现有实现或组件入口
- 优先复用现有 `*Backend.js` 文件命名、请求参数格式和 `FetchFilter.js` 逻辑。
- 相同资源的接口优先集中在同一个 backend 文件里。

## 禁止事项
- 不要把页面状态或 UI 提示逻辑混进 backend 模块。
- 不要为相似资源重复写一套几乎相同的请求封装。

## 本目录最小验证命令
- `cd web && yarn fix`
- 如变更请求入口较多，再执行 `cd web && yarn build` 检查打包。

## 与上级规则的继承或覆盖说明
- 继承 `web/src/AGENTS.md`；本目录额外强调“请求封装集中化”。
- 仅在前端 API 层覆盖上级。
