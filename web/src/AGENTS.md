# AGENTS.md

## 本目录职责
- 维护前端主代码目录，包括页面、共享组件、请求封装和业务展示逻辑。
- 这里是前端行为和 UI 的主战场，默认应遵循现有 Ant Design + MD2 偏好组合。

## 允许修改范围
- 允许修改页面、组件、样式、请求封装和本目录的子模块规则。
- 新增功能优先接入已有页面骨架、`backend` 请求层和 `common` 组件层。

## 必须复用的现有实现或组件入口
- 优先复用现有列表页/编辑页结构、`Conf.js` 配置、`backend/` 请求模块与 `common/` 共享组件。
- 新增 i18n key 时同步更新 `web/src/locales/en/data.json`。

## 禁止事项
- 不要把请求细节散落到每个页面组件。
- 不要为了追求新风格推翻现有 Ant Design 页面结构。

## 本目录最小验证命令
- `cd web && yarn fix`
- `cd web && yarn test --watchAll=false`
- 影响路由或打包时再执行 `cd web && yarn build`。

## 与上级规则的继承或覆盖说明
- 继承 `web/AGENTS.md` 与根规则；本目录额外强调“页面、组件、请求三层分离”。
- 子目录如有更近规则，优先使用更近规则。
