# AGENTS.md

## 本目录职责
- 维护部署相关工具与部署测试。
- 这里更偏交付与环境验证，不承载核心业务逻辑。

## 允许修改范围
- 允许修改部署帮助逻辑、部署测试和与打包流程直接相关的代码。
- 若变更影响构建入口，要同步检查 `build.sh`、`Dockerfile` 等外围文件。

## 必须复用的现有实现或组件入口
- 优先复用现有部署测试模式和现有构建入口。
- 不要重新定义与主程序不同的启动语义。

## 禁止事项
- 不要把业务实现搬到部署目录。
- 不要引入只为当前机器环境成立的硬编码路径。

## 本目录最小验证命令
- `go test ./deployment/...`
- 如影响打包流程，可额外执行 `go build ./...`。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“环境无关和可部署性”。
- 仅补充部署层边界。
