# AGENTS.md

## 本目录职责
- 维护身份提供商接入、OAuth/OIDC/SAML 等第三方登录适配。
- 重点是 provider 差异收敛与用户信息映射。

## 允许修改范围
- 允许修改 provider 实现、用户信息转换和与登录流程直接相关的局部辅助函数。
- 新增 IdP 时优先沿用现有 provider 注册和字段映射模式。

## 必须复用的现有实现或组件入口
- 优先复用 `provider.go`、已有 token/userinfo 获取流程和错误处理方式。
- 与账户创建或绑定相关的最终规则优先复用 `object/` 与 `controllers/` 调用链。

## 禁止事项
- 不要在本目录散落页面逻辑或无关的本地账户策略。
- 不要复制已有 IdP 的完整实现再做小改。

## 本目录最小验证命令
- `go test ./...`
- 如改动高风险登录链路，至少补查受影响 provider 的测试或调用方。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“第三方协议兼容和字段映射一致性”。
- 仅在 IdP 集成层覆盖上级。
