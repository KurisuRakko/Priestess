# AGENTS.md

## 本目录职责
- 维护核心领域对象、业务规则、初始化流程、同步任务和大部分共享能力。
- 这里是后端业务真正规则层，任何跨控制器复用都优先在这里落地。

## 允许修改范围
- 允许修改对象模型、领域规则、对象间协作、初始化与缓存、定时任务和与对象相关测试。
- 影响较广的改动要优先在这里收敛，而不是让多个控制器各自打补丁。

## 必须复用的现有实现或组件入口
- 优先复用已有对象函数、缓存、provider/item 模式和测试样例。
- 新增资源前先搜索相邻对象是否已有近似字段、状态或生命周期处理。

## 禁止事项
- 不要把 HTTP 请求解析、页面展示或 CLI 细节混入领域层。
- 不要在多个对象文件里各自复制一段相同业务判断。

## 本目录最小验证命令
- `go test ./object/...`
- 高影响改动再补跑 `go test ./...` 以检查跨包回归。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“领域逻辑单一来源”。
- 若与上级冲突，优先维持对象层作为业务真相来源。
