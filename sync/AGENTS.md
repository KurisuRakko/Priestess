# AGENTS.md

## 本目录职责
- 维护现有同步链路、数据库同步和同步工具函数。
- 重点是同步正确性、增量行为和异常恢复。

## 允许修改范围
- 允许修改同步实现、同步工具、状态处理与对应测试。
- 如果同步逻辑依赖对象规则，优先调用 `object/syncer*` 现有能力。

## 必须复用的现有实现或组件入口
- 优先复用已有同步接口、数据库适配和状态跟踪方式。
- 新增同步源时先跟随现有主从或 provider 抽象。

## 禁止事项
- 不要在同步层偷偷重写对象规则。
- 不要通过复制旧同步流程实现新版本功能。

## 本目录最小验证命令
- `go test ./sync/...`
- 高影响改动再补跑 `go test ./object/...`。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“同步正确性优先于局部便利”。
- 仅在同步语义层补充规则。
