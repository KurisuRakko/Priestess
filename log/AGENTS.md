# AGENTS.md

## 本目录职责
- 维护日志提供商、系统日志与平台差异化日志能力。
- 重点是平台兼容、日志级别和输出稳定性。

## 允许修改范围
- 允许修改日志适配、平台特定实现和与日志直接相关的局部工具。
- 新增平台差异逻辑时优先保持统一入口不变。

## 必须复用的现有实现或组件入口
- 优先复用已有 provider 选择方式和平台分文件模式。
- 跨层日志配置优先跟随 `conf/` 与启动初始化逻辑。

## 禁止事项
- 不要把业务分支藏在日志层。
- 不要改变日志级别语义而不检查调用方预期。

## 本目录最小验证命令
- `go test ./...`
- 如改动平台条件编译文件，至少确认相关平台文件能编译。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“输出稳定和平台兼容”。
- 仅在日志实现层覆盖上级。
