# AGENTS.md

## 本目录职责
- 维护服务级编排、站点代理协作和对外服务流程。
- 这里适合承载跨多个对象但又不属于 HTTP 控制器的流程。

## 允许修改范围
- 允许修改服务编排、跨对象调用顺序和服务层辅助函数。
- 如果逻辑已经变成对象级规则，应继续下沉到 `object/`。

## 必须复用的现有实现或组件入口
- 优先复用已有站点、代理、oauth 等服务流程入口。
- 新增编排前先搜索是否已有近似流程可以扩展。

## 禁止事项
- 不要在服务层重复定义对象层真相。
- 不要把纯 HTTP 请求解析或前端展示逻辑放进来。

## 本目录最小验证命令
- `go test ./service/...`
- 如服务编排影响广，再补跑 `go test ./object/...` 或 `go test ./...`。

## 与上级规则的继承或覆盖说明
- 继承根规则；本目录额外强调“编排层不成为第二个业务真相层”。
- 仅在服务编排边界上覆盖上级。
